# 宸观 BrightEye · 宸宇护目，智能时长管控护眼伴侣系统（参赛原型 Demo · v1.16.0）

> 开源仓库：<https://github.com/Ixnery-Karity/ChenGuan_BrightEye>（仅软件代码，
> 不含商业计划书等竞赛文档与演示包）。

面向长时间用眼人群的**非接触式 AI 视觉健康监测**桌面应用。通过普通摄像头
实时分析**眨眼频率、用眼距离、坐姿（颅椎角/高低肩）、表情情绪**，结合
**20-20-20 护眼法则**给出分级健康建议；原创桌宠「弥悠」可接入**大模型自然对话**
（好感度养成 + 分层记忆跨会话持久化），复盘报告提供 **AI 行为洞察**，
监测历史自动入 SQLite 库并支持**周报/月报**，极端用眼时触发**强制护眼干预**，
会话结束生成可视化健康报告。

> 版本记录见 [`CHANGELOG.md`](CHANGELOG.md)；大模型部署见
> [`docs/大模型接入与部署指南.md`](docs/大模型接入与部署指南.md)。
> **离线优先铁律**：不装任何大模型也能完整演示（聊天走离线脚本、复盘走规则
> 建议、情绪走本地表情系数），大模型是「可选增强」，失败自动安全回退。

> 合规说明：本产品仅提供用眼/体态/情绪**行为引导与提示**，不构成医疗诊断、
> 治疗或用药处方。涉及药物处只做"建议咨询医生/药师"的提示。

## 一、快速运行（任何机器，零摄像头也能演示）

本机若未安装摄像头或 mediapipe，程序会**自动回退到内置模拟器**，完整跑通
「监测 → 实时建议 → 健康报告」全链路，可直接用于答辩演示。

### 完整安装包（v1.16.0 · 软件成品形态）

- 运行 `python -m brighteye.tools.build_exe` 一键产出
  **`dist_installer/宸观BrightEye_Setup_v1.16.0-demo.exe`** 中文安装向导
  （PyInstaller onedir 打包后自动调用 Inno Setup 编译）。
- 免管理员安装（默认装到 `%LOCALAPPDATA%\Programs`），自动创建桌面/
  开始菜单快捷方式；安装版免 Python 环境双击即用，用户数据与报告按
  Windows 惯例落 `%LOCALAPPDATA%\ChenguanBrightEye\`。
- **双击卸载**（v1.16.0）：开始菜单含「卸载 宸观 BrightEye」入口，
  也可走系统「设置 → 应用」卸载；卸载时询问是否连监测历史/报告一并
  删除，**默认保留**——重装后个人健康档案不丢。

### 双击启动（源码运行 · 无命令行窗口）

- 双击 **`brighteye/启动宸观BrightEye.vbs`** —— 以 pythonw 无窗口启动，
  全程不出现任何命令提示符；**加载页点开即见**（v1.15.0 立绘改后台
  异步抠图 + 首帧强制上屏，启动不再黑等，退出也从半分钟改为秒退）。
  **v1.16.0 起自动检测运行环境**：没装 Python 或缺库时弹窗提示一键安装
  （调用 `安装运行环境.bat` 自动装 Python 3.12 + 全部依赖，含清华镜像
  兜底），装完自动继续启动——全新机器双击即用。
- 双击 **`brighteye/创建桌面快捷方式.vbs`** —— 在桌面生成带 Q 版弥悠图标的
  「宸观 BrightEye」快捷方式（v1.15.0 修复图标路径转义损坏并自动刷新
  系统图标缓存，桌面图标正确显示 Q 版弥悠）。
- 运行期右下角**系统托盘**常驻宸观图标（v1.15.0，纯 ctypes 零依赖）：
  左键打开仪表盘，右键菜单切四模式 / 和弥悠聊天 / 退出；
  关闭窗口=收起为桌宠，进程由托盘图标承接。

### 命令行启动（开发/调参）

```bash
# 进入项目根目录（challenge/）
cd challenge

# 1) 图形仪表盘（推荐演示）—— 自动选择后端 + 悬浮桌宠「弥悠」常驻陪伴
python -m brighteye.main

# 2) 强制模拟数据（无摄像头时）
python -m brighteye.main --simulate

# 3) 加速演示：8 倍速快速触发"久用眼/前倾"告警
python -m brighteye.main --simulate --fast 8

# 4) 指定启动模式（陪伴 / 严格 / 复盘 / 勿扰）
python -m brighteye.main --mode strict

# 5) 启动即收起为悬浮桌宠，仅留小窗陪伴（轻打扰日常模式）
python -m brighteye.main --pet

# 6) 无界面自测：跑 20 秒后直接生成 HTML 报告（适合录屏/CI）
python -m brighteye.main --simulate --headless 20

# 7) 视觉推理独立子进程（性能隔离，规避 GIL 掉帧；失败自动回退）
python -m brighteye.main --mp-vision

# 8) 局域网多端同步：手机上报用眼时长，报告合并全天候负荷（默认端口 8765）
python -m brighteye.main --sync

# 9) 周报 / 月报：聚合 SQLite 历史会话直接出报告（无需摄像头，v1.11.0）
python -m brighteye.main --report weekly
python -m brighteye.main --report monthly
```

报告输出在 `brighteye/reports/` 目录（HTML，可直接截图进 PPT）；监测历史
与偏好在 `brighteye/data/`。**v1.14.0 起两目录锚定包路径**，从任何目录
启动都不会散落多套数据；安装版（frozen）则按 Windows 惯例落
`%LOCALAPPDATA%\ChenguanBrightEye\`（v1.15.0）。

### 悬浮桌宠「弥悠」（v1.11.0 原创人设，见 `docs/弥悠人设.md`）
宸观视觉引擎的拟人化中枢：浅粉紫长发·半睁紫瞳·猫耳·黑色项圈（=本地隐私锁）·
黑色桃心发夹·温柔慵懒看护者；「视觉负荷共鸣」——你不眨眼久用眼，她就替你的
眼睛犯困喊累。支持外部 galgame 差分立绘 + 程序化矢量兜底，无边框透明置顶可拖拽：
**左键点开仪表盘、右键切换模式/退出、按住拖动位置**；情绪随用眼状态变色，
气泡台词实时陪伴。

### 大模型增强能力（v1.8.1 · 全部可选，离线自动回退）
| 能力 | 说明 | 离线兜底 |
| --- | --- | --- |
| 🗣 **桌宠大模型对话** | 弥悠接入本地 Ollama / 云端 API（默认 `qwen2.5:7b-instruct`），多轮短期记忆 + 中期 SQLite 关键事件记忆回注 + 实时用眼/情绪上下文注入，好感度档位语气随关系进阶；好感度增量仍由离线规则判定防刷分（单日上限 + 日衰减 + 跨会话持久化）。 | 内置温柔关键词脚本 |
| 🧠 **AI 行为洞察报告** | 复盘报告用推理模型（默认 `deepseek-r1:7b`）基于结构化指标产出「行为习惯洞察 + 个性化建议」；自动剥离 R1 思维链。 | 规则版个性化建议 |

| ⛔ **极端用眼强制干预** | 连续用眼久 + 距离过近 + 眨眼率极低同时持续满足才触发（带冷却）；soft=全屏遮罩+强制休息倒计时（默认），hard=Windows 系统锁屏。 | 始终本地可用 |

> （兼容 DeepSeek 官方等 OpenAI 风格 API）。**v1.12.0 起软件启动会自动拉起本地
> Ollama 服务**（探测不到时后台无窗口启动 `ollama serve`，≤25s 就绪，失败静默降级
> 离线；`config.llm.auto_start_ollama` 可关）。模型 8.8GB 超 GitHub 上传限制，
> 仓库只含安装脚本不含模型本体，见 `llm_models/README.md`。

## 二、启用真实摄像头检测（实时）

```bash
pip install opencv-python mediapipe   # 已在 Python 3.14 + mediapipe 0.10.35 实测可用
python -m brighteye.main              # 检测到摄像头即自动启用 MediaPipe 实时检测
python -m brighteye.main --real       # 强制实时检测，启用失败直接报错(不回退模拟)
python -m brighteye.main --camera 1   # 指定摄像头索引(多摄像头时)
```

> 实时检测基于 MediaPipe **Tasks API**，需 `.task` 模型文件，已随仓库放在
> `brighteye/assets/models/`（face_landmarker.task、pose_landmarker_lite.task）。
> 程序启动时会打印实际数据源；若回退模拟会给出原因提示。
> **v1.14.0 起单进程模式也用独立采样线程**：采集+推理与 UI 解耦，按
> `fps_target` 固定节奏采样——UI 卡顿不再造成眨眼漏检，推理不再拖累界面；
> 距离/颅椎角实时值改用近 3 秒短窗（姿势变化秒级上屏），摄像头缓冲压到
> 1 帧消除画面滞后。性能根因分析见 `docs/启动与卡顿性能分析.md`。

**核心算法**
- 眨眼：FaceMesh 6 点眼纵横比 `EAR=(‖p2-p6‖+‖p3-p5‖)/(2‖p1-p4‖)`，
  连续低于阈值帧数判定一次眨眼；低于 10 次/分提示干眼风险。
- 距离：`distance = focal × 平均瞳距 / 像素瞳距`，估算用眼距离。
- 坐姿：Pose 关键点按**同侧拓扑**（左耳配左肩/右耳配右肩，可见度加权，
  Gemini 交叉评审建议合入）计算颅椎角(CVA)近似与双肩倾角，判定低头/前倾/高低肩。
- 情绪：52 维 blendshapes → FACS 动作单元(AU) → Ekman 情绪原型打分，
  EMA 平滑 + 迟滞去抖，输出 积极/平静/疲惫/压力/低落（可解释、无需训练）。
- 大模型：`llm_client` 仅用标准库 urllib，自动探测 OpenAI 兼容 API → 本地
  Ollama → 离线兜底；聊天用 Qwen2.5-Instruct，复盘分析用 DeepSeek-R1（剥离思维链）。
