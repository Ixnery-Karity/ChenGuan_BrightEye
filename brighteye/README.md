# 宸观 BrightEye · 宸宇护目，智能时长管控护眼伴侣系统（参赛原型 Demo · v1.10.0）

> 开源仓库：<https://github.com/Ixnery-Karity/ChenGuan_BrightEye>（仅软件代码，
> 不含商业计划书等竞赛文档与演示包）。

面向长时间用眼人群的**非接触式 AI 视觉健康监测**桌面应用。通过普通摄像头
实时分析**眨眼频率、用眼距离、坐姿（颅椎角/高低肩）、表情情绪**，结合
**20-20-20 护眼法则**给出分级健康建议；桌宠「文乃」可接入**大模型自然对话**，
复盘报告提供 **AI 行为洞察**，极端用眼时触发**强制护眼干预**，会话结束生成
可视化健康报告。

> 版本记录见 [`CHANGELOG.md`](CHANGELOG.md)；大模型部署见
> [`docs/大模型接入与部署指南.md`](docs/大模型接入与部署指南.md)。
> **离线优先铁律**：不装任何大模型也能完整演示（聊天走离线脚本、复盘走规则
> 建议、情绪走本地表情系数），大模型是「可选增强」，失败自动安全回退。

> 合规说明：本产品仅提供用眼/体态/情绪**行为引导与提示**，不构成医疗诊断、
> 治疗或用药处方。涉及药物处只做"建议咨询医生/药师"的提示。

## 一、快速运行（任何机器，零摄像头也能演示）

本机若未安装摄像头或 mediapipe，程序会**自动回退到内置模拟器**，完整跑通
「监测 → 实时建议 → 健康报告」全链路，可直接用于答辩演示。

```bash
# 进入项目根目录（challenge/）
cd challenge

# 1) 图形仪表盘（推荐演示）—— 自动选择后端 + 悬浮桌宠「文乃」常驻陪伴
python -m brighteye.main

# 2) 强制模拟数据（无摄像头时）
python -m brighteye.main --simulate

# 3) 加速演示：8 倍速快速触发"久用眼/前倾"告警
python -m brighteye.main --simulate --fast 8

# 4) 指定启动模式（陪伴 / 严格 / 复盘 / 勿扰）
python -m brighteye.main --mode strict

# 5) 启动即收起为悬浮桌宠，仅留小窗陪伴（轻打扰日常模式）
python -m brighteye.main --pet

# 6) 无界面自测：跑 20 秒后直接生成 HTML 报告（适合录屏/CI）
python -m brighteye.main --simulate --headless 20

# 7) 视觉推理独立子进程（性能隔离，规避 GIL 掉帧；失败自动回退）
python -m brighteye.main --mp-vision

# 8) 局域网多端同步：手机上报用眼时长，报告合并全天候负荷（默认端口 8765）
python -m brighteye.main --sync
```

报告输出在 `reports/` 目录（HTML，可直接截图进 PPT）。

### 四种运行模式（顶部一键切换 / 桌宠右键切换）
| 模式 | 说明 |
| --- | --- |
| 🐾 陪伴模式 | 默认。悬浮桌宠「文乃」低打扰陪伴，傲娇台词提醒。|
| 🔥 严格模式 | 不良用眼习惯一出现即弹窗；持续不改 → 严重度逐级升级，**颜色由青绿→琥珀→橙→警示红**逐级加深。|
| 📊 复盘模式 | 深度监测，每 20 分钟自动生成一份完整健康报告。|
| 🌙 勿扰模式 | 仅后台记录，零弹窗零台词。|

### 悬浮桌宠「文乃」
程序化矢量绘制（暗金发·翠绿瞳·红缎带·虎牙·猫耳猫尾·傲娇人设），
无边框透明置顶可拖拽：**左键点开仪表盘、右键切换模式/退出、按住拖动位置**；
情绪随用眼状态变色（健康绿稳、告警红快），气泡台词实时陪伴。

## 二、启用真实摄像头检测（实时）

```bash
pip install opencv-python mediapipe   # 已在 Python 3.14 + mediapipe 0.10.35 实测可用
python -m brighteye.main              # 检测到摄像头即自动启用 MediaPipe 实时检测
python -m brighteye.main --real       # 强制实时检测，启用失败直接报错(不回退模拟)
python -m brighteye.main --camera 1   # 指定摄像头索引(多摄像头时)
```

> 实时检测基于 MediaPipe **Tasks API**，需 `.task` 模型文件，已随仓库放在
> `brighteye/assets/models/`（face_landmarker.task、pose_landmarker_lite.task）。
> 程序启动时会打印实际数据源；若回退模拟会给出原因提示。

```
│   ├── 多端数据同步API.md        局域网同步接口定义与手机端对接

├── tools/
│   └── build_exe.py     一键打包脚本（PyInstaller onedir + 生成 Inno Setup 脚本）
├── vision/
│   ├── detectors.py     MediaPipe 真实检测后端（EAR 眨眼 / 同侧拓扑 CVA 坐姿 / 瞳距测距 / blendshapes）
│   └── worker.py        视觉推理独立子进程（spawn 安全，Queue 回传，崩溃自动回退）
├── core/
│   ├── monitor.py       编排器：后端选择 + 指标 + 计时 + 建议 + 模式/情绪/守护/游戏勿扰
│   ├── metrics.py       会话指标采集与滚动统计（含情绪时间线）
│   ├── blink_counter.py EAR→眨眼事件状态机
│   ├── simulator.py     无摄像头时的合成数据后端（含渐趋疲惫表情系数）
│   ├── rest_timer.py    20-20-20 与连续用眼计时
│   ├── advice_engine.py 规则化分级建议（合规：不诊断不开药）
│   ├── modes.py         四模式 + 严格模式逐级升级器（颜色加深）
│   ├── persona.py       桌宠「文乃」台词与情绪系统（傲娇人设 + 情绪关怀台词）
│   ├── llm_client.py    统一大模型客户端（唯一入口，仅标准库 urllib，零重依赖）
│   ├── emotion.py       表情情绪估计（FACS-AU → Ekman 情绪原型 + 关怀优先级仲裁）
│   ├── guardian.py      极端用眼强制干预（soft 遮罩 / hard 系统锁屏）
│   ├── system_watch.py  全屏游戏检测(ctypes) + DDC/CI 显示器亮度控制(可选)
│   ├── sync.py          局域网多端同步服务（纯标准库 HTTP，聚合指标不传画面）
│   ├── chat_engine.py   桌宠对话引擎（大模型多轮记忆 + 离线傲娇脚本回退）
│   ├── report_charts.py 报告图表四件套（纯标准库内联 SVG：雷达/趋势/热力/建议）
│   └── health_report.py 健康评分 + 文本/HTML 报告（图表四件套 + AI 行为洞察 + 跨设备合并）
└── ui/
    ├── app.py           Tkinter 实时仪表盘（电竞导播风 + 粒子背景 + 模式切换 + 强制遮罩）
    ├── chat.py          galgame 风桌宠聊天窗口
    ├── particles.py     Canvas 粒子背景（星座连线 + 状态变色 + 帧耗时自适应降载）
    └── pet.py           悬浮桌宠「文乃」（透明置顶可拖拽 + 程序化动画形象）
```

**核心算法**
- 眨眼：FaceMesh 6 点眼纵横比 `EAR=(‖p2-p6‖+‖p3-p5‖)/(2‖p1-p4‖)`，
  连续低于阈值帧数判定一次眨眼；低于 10 次/分提示干眼风险。
- 距离：`distance = focal × 平均瞳距 / 像素瞳距`，估算用眼距离。
- 坐姿：Pose 关键点按**同侧拓扑**（左耳配左肩/右耳配右肩，可见度加权，
  Gemini 交叉评审建议合入）计算颅椎角(CVA)近似与双肩倾角，判定低头/前倾/高低肩。
- 情绪：52 维 blendshapes → FACS 动作单元(AU) → Ekman 情绪原型打分，
  EMA 平滑 + 迟滞去抖，输出 积极/平静/疲惫/压力/低落（可解释、无需训练）。
- 大模型：`llm_client` 仅用标准库 urllib，自动探测 OpenAI 兼容 API → 本地
  Ollama → 离线兜底；聊天用 Qwen2.5-Instruct，复盘分析用 DeepSeek-R1（剥离思维链）。

## 四、可演示的卖点（对应评分细则"创新性/商业性"）
1. 纯软件、非接触、隐私友好（可本地推理，画面与大模型均可全程不出本机）。
2. 多维度健康量化（用眼 + 坐姿 + 情绪）+ 综合评分，形成可追踪的健康档案。
3. 阈值全部可配置，支持按个体/场景（学生、程序员、设计师）校准。
4. **大模型陪伴 + AI 行为洞察**：桌宠可深聊、复盘报告给个性化归因建议。
5. **情绪管理·心理健康呵护**：本地表情情绪识别 + 主动关怀台词，对齐商业计划书卖点。
6. **极端用眼强制干预**：多条件持续触发，soft 遮罩 / hard 锁屏两档保护。
7. B 端可对接学校/眼科诊所/企业工会，C 端订阅 + 硬件联动。
