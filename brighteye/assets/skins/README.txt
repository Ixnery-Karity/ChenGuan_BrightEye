宸观 BrightEye · 皮肤上传规则（v1.17.0）
================================================================

一、目录结构（一套皮肤 = 一个独立文件夹）
  assets/skins/<皮肤名>/
      idle.png          必需（基准立绘，缺它整套皮肤不生效）
      happy.png         可选（开心/表扬）
      pout.png          可选（轻度担心/提醒）
      angry.png         可选（严重担心/警告）
      sleepy.png        可选（犯困/离座）
      blink.png         可选（通用闭眼帧，与 idle 同姿势）
      happy_blink.png / pout_blink.png / angry_blink.png  可选（各情绪闭眼差分）

二、激活方式
  config.py → AppConfig.skin = "<皮肤名>"（默认 "miyou"）。
  该目录存在 idle.png 即整套生效；缺失的情绪自动用本皮肤 idle 帧代替。

三、隔离铁律（防新旧形象冲突）
  1) 皮肤生效时**只**从该皮肤文件夹取图，绝不回头混用旧
     assets/pet/wenna_*.png（文乃旧立绘）——避免半新半旧串戏。
  2) 旧文乃立绘仅在「没有任何有效皮肤」时作为整套兜底；再无则矢量绘制。
  3) 上传新皮肤 = 新建一个文件夹，不覆盖、不修改既有皮肤目录。

四、图片要求
  · 竖版全身立绘，高度 ≥ 768px；PNG 格式。
  · 透明底最佳；纯色底（白/灰）也可——程序会自动 floodfill 抠除背景。
  · 同套差分务必同一角色同一画风（AI 出图锁同一 seed / 用图生图保一致）。

五、本仓库自带皮肤
  miyou/   弥悠官方皮肤（本地 SD WebUI + anything-v5 生成，seed=3721，
           提示词见 agent 生成脚本；符合 docs/弥悠人设.md：粉紫长发/
           半睁紫瞳/猫耳/黑项圈/桃心发夹/猫爪防线姿势）。
