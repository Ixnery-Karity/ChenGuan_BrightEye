# 宸观 BrightEye · 大模型接入与部署指南

> 面向三处能力：①桌宠「文乃」对话、②复盘/健康报告的行为习惯洞察、③表情情绪呵护。
> 设计铁律：**离线优先**——不装任何大模型也能完整演示（聊天走离线脚本、复盘走规则建议、
> 情绪走本地表情系数），大模型是「可选增强」，任何环节失败都安全回退，绝不阻塞界面。

---

## 一、总体架构：混合（离线兜底 + 可选 LLM，自动探测）

```
桌宠聊天 chat_engine ─┐
                      ├─► core/llm_client.py（唯一入口，仅用标准库 urllib）
复盘洞察 health_report ┘        │
                                ├─ ① OpenAI 兼容 API（读环境变量，含 DeepSeek 官方）
                                ├─ ② 本地 Ollama（localhost:11434）
                                └─ ③ 都没有 → 返回 None → 调用方回退离线逻辑
表情情绪 emotion ── 复用 MediaPipe FaceLandmarker 的 52 维 blendshapes（本地、零新增模型）
```

- **零重依赖**：LLM 调用不引入 `openai`/`requests`，只用 Python 标准库 `urllib`，保持「任意机器可演示」。
- **模型分工**：
  - 聊天 = **Qwen2.5-Instruct**（指令跟随/角色扮演自然，傲娇人设不易崩）；
  - 复盘分析 = **DeepSeek-R1-Distill**（推理能力强，擅长从指标归因；`<think>` 思维链由 `strip_think` 自动剥离）。
- 配置集中在 `brighteye/config.py` 的 `LLMConfig`（`enabled / chat_model / analysis_model / base_url / api_key_env / ollama_host / timeout_sec`）。

## 二、三种部署方案对比

| 方案 | 适用场景 | 隐私 | 成本 | 配置难度 |
|------|----------|------|------|----------|
| **A. 本地 Ollama** | 有一定显存/内存的本机、注重隐私、离线可用 | ★★★ 全本地不出网 | 免费 | 中（装 Ollama + 拉模型） |
| **B. 云端 API（OpenAI 兼容/DeepSeek 官方）** | 机器性能弱、要最好效果、可联网 | ★ 数据经第三方 | 按量付费 | 低（设两个环境变量） |
| **C. 纯离线兜底（默认）** | 演示/答辩/断网 | ★★★ 全本地 | 免费 | 零（开箱即用） |

> 竞赛现场建议：**默认走 C**（稳、快、断网也能演示）；若想展示大模型效果，
> 提前用 **A** 在本机拉好模型，或临时用 **B** 配一个 Key。三者随时可切，互不影响。

---

## 三、方案 A：本地 Ollama（推荐用于隐私 / 离线增强）

1. 安装 Ollama：访问 https://ollama.com 下载对应系统安装包并安装。
2. 启动服务（安装后通常自启，监听 `http://localhost:11434`）：
   ```bash
   ollama serve
   ```
3. 拉取模型（首次较大，数 GB，耐心等待）：
   ```bash
   ollama pull qwen2.5:7b-instruct     # 桌宠聊天
   ollama pull deepseek-r1:7b          # 复盘分析
   ```
   > 机器较弱可换更小档：`qwen2.5:3b-instruct`、`deepseek-r1:1.5b`，并在 `config.py` 同步模型名。
4. 无需设任何环境变量——`llm_client` 会自动探测 `localhost:11434` 的 `/api/tags`，
   探测成功即启用。确认 `LLMConfig.enabled = True`（默认）即可。

---

## 四、方案 B：云端 OpenAI 兼容 API（含 DeepSeek 官方）

只需设两个环境变量（示例为 DeepSeek 官方端点）：

**Windows（PowerShell）**
```powershell
setx BRIGHTEYE_LLM_BASE "https://api.deepseek.com/v1"
setx BRIGHTEYE_LLM_KEY  "sk-你的密钥"
```
**macOS / Linux（bash）**
```bash
export BRIGHTEYE_LLM_BASE="https://api.deepseek.com/v1"
export BRIGHTEYE_LLM_KEY="sk-你的密钥"
```

- 兼容任意 OpenAI 风格 `/chat/completions` 端点（硅基流动、OpenRouter、自建等同理，改 `BASE` 即可）。
- 也兼容 `OPENAI_BASE_URL` / `OPENAI_API_KEY` 两个通用变量名。
- 云端场景下聊天/复盘用的具体模型名，按你所用服务商填入 `config.py` 的 `chat_model` / `analysis_model`。

---

## 五、方案 C：纯离线兜底（默认，无需任何操作）

不装 Ollama、不设环境变量时：
- 桌宠聊天 → 走内置傲娇关键词脚本（`chat_engine` 的 `_REPLIES`），`ChatTurn.is_llm=False`；
- 复盘报告 → 「🧠 AI 行为洞察」段落自动隐藏，展示规则版个性化建议；
- 表情情绪 → 仍然可用（基于本地 MediaPipe blendshapes，与 LLM 无关）；
- 极端用眼强制干预 → 仍然可用（纯本地判定 + tkinter 遮罩/系统锁屏）。

如需**强制纯离线**（即使本机装了 Ollama 也不调用），把 `config.py` 的 `LLMConfig.enabled` 设为 `False`。

## 六、表情情绪分析（本地、零新增模型，v1.8 升级为 FACS-AU 引擎）

- 复用已在用的 `face_landmarker.task`，在 `detectors.py` 创建时加 `output_face_blendshapes=True`，
  得到 52 维表情系数（微笑/皱眉/眯眼/张口/眨眼/内外眉等）。
- `core/emotion.py` 采用**可解释三级管线**（比早期单阈值判定更准、更稳）：
  1. **blendshapes → FACS 动作单元(AU)**：把 52 维系数归约为约 13 个 AU
     （AU4 皱眉、AU7 眼睑收紧、AU12 嘴角上扬、AU15 嘴角下拉、AU1 内眉上扬、
     AU6 脸颊上提、AU24 抿唇、AU26 张口、AU45 眨眼加重…），左右对称部位取均值；
  2. **AU → Ekman 情绪原型加权打分**：每类情绪 = 相关 AU 的加权组合（正促成/负抑制），
     依 FACS 情绪编码惯例设权重，输出 `积极 / 平静 / 疲惫 / 压力 / 低落` 五类；
  3. **EMA 平滑 + 迟滞(hysteresis)去抖**：AU 通道逐帧 EMA，情绪切换需高出当前情绪
     `switch_margin` 裕度才生效，避免逐帧跳变；低于 `neutral_bias` 基线判为「平静」。
- 相比单阈值法，此法用**多 AU 组合**区分「压力(皱眉+抿唇)」与「疲惫(眨眼加重+哈欠)」，
  用 `browInnerUp+mouthFrown` 识别「低落」，显著降低误判；全程无需训练、可逐条溯源到肌肉动作，
  非常适合答辩讲解。参数集中在 `config.py` 的 `EmotionConfig`。
- 关怀触发加了**置信门限** `care_min_score`：负面情绪不仅要持续，还要打分足够高才让文乃主动安慰，
  进一步防误触。
- 检测到「疲惫/压力/低落」持续一段时间且置信足够，文乃以**聊天关怀台词**主动安慰
  （对齐商业计划书「情绪管理·心理健康呵护」）。
- 无摄像头的 `--simulate` 演示下，模拟器会随用眼时长合成「渐趋疲惫」的表情
  （积极→疲惫→压力），便于现场展示该能力。

### 6.1 可选进阶：用数据集训练 ML 情绪模型（非必需，保持离线铁律）

当前 FACS-AU 引擎已能稳定演示且零依赖。若追求更高精度，可在**不破坏「离线优先」**
前提下叠加一个轻量分类器（默认关闭，训练好才启用）：

1. **数据集**：`FER2013`（35887 张 48×48 表情图，7 类）最常用；或 `AffectNet`、`RAF-DB`。
   注意 FER2013 是原始像素图，需先过一遍 MediaPipe 提取 blendshapes，形成「52 维 → 情绪」样本。
2. **特征**：直接用本项目已产出的 52 维 blendshapes（或归约后的 AU）作为特征，
   **无需再训一个 CNN**，避免引入 GPU/大模型依赖——这正是 blendshapes 的价值。
3. **模型**：`scikit-learn` 的 `SVM` / `RandomForest`（几十 KB、CPU 毫秒级推理，
   与「任意机器可演示」铁律兼容）；公开工作里 blendshapes+SVM 可达可用精度，
   LSTM 在 FER2013 上约 ~71% 准确率可作参考上限。
4. **接入（预留设计，当前未接线）**：训练产物存为 `assets/models/emotion_clf.pkl`；
   在 `EmotionEstimator` 里加一个可选分支——启动探测到该文件则用分类器输出、否则回退
   现有 FACS-AU 规则（与 LLM 一样「有则增强、无则兜底」）。当前版本尚未内置该分支，
   保持零依赖；需要时按此思路补一段 `try: import joblib` 的软加载即可。
5. **映射**：FER2013 的 7 类（angry/disgust/fear/happy/sad/surprise/neutral）需归并到本项目
   5 类（如 angry+fear→压力、sad→低落、happy→积极、neutral→平静，疲惫仍靠 AU45/AU26 规则补充，
   因表情数据集通常无「疲惫」标签）。

> 结论：ML 路径是**可选增益**而非必需项。答辩现场默认走 FACS-AU 规则引擎（稳、快、可解释）；
> 训练好分类器后放入 `assets/models/` 即自动启用，不改变部署方式。

## 七、极端用眼强制干预（护眼守护）

- `core/guardian.py`：当**连续用眼过久 + 距离持续过近 + 眨眼率极低**同时满足并持续若干秒，
  触发一次强制干预（带冷却时间防误触，阈值集中在 `config.py` 的 `GuardianConfig`）。
- 两档强度（`guardian.mode`）：
  - **soft（默认）**：全屏半透明遮罩 + 文乃立绘 + 关怀台词 + 强制休息倒计时，倒计时结束才可关闭；纯 UI、零系统风险。
  - **hard**：调用 Windows `LockWorkStation()` 真系统锁屏；非 Windows 自动降级 soft。

## 八、两个外部目录说明（评估结论）

- `model/ui-ux-pro-max-skill-main`：是一套 **UI/UX 设计用的 Claude 技能**（数十种界面风格 + 设计规则），
  **不是可运行的模型**。用途：可作为美化聊天窗口/仪表盘视觉的设计参考，非必需。
- `model/claude-code-source`：是 **Claude Code 自身的源码**（含 voice/query/services 等），
  **不是模型**。用途：LLM 流式客户端写法的参考，非必需。
- 结论：两者都**不能直接当模型用**；真正的大模型能力走上文方案 A（Ollama）或 B（API）。

## 九、快速自检

```bash
# 离线兜底（不装任何 LLM）——应一切正常，聊天 is_llm=False、报告无 AI 洞察段
python -m brighteye.main --simulate

# 启用 LLM 后——聊天出现自然多轮对话、报告出现「🧠 AI 行为洞察」段落
# （装好 Ollama 并 pull 模型，或设好 API 环境变量后再次运行）
```
