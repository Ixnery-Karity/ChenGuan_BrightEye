```
┌──────────────────────────────────────┴──────────────────────────────────────────────┐
│ core 核心层：monitor 编排 · metrics 指标 · blink_counter · emotion 情绪 · advice 建议  │
│ rest_timer 20-20-20 · guardian 极端干预 · persona/chat_engine 桌宠人格 · history 存储 │
│ health_report/period_report/report_charts 报告 · llm_client 大模型 · sync 多端同步    │
└──────────────────────────────────────┬──────────────────────────────────────────────┘
                                       │ FrameSample（逐帧原始检测值）
┌──────────────────────────────────────┴──────────────────────────────────────────────┐
│ vision 视觉层：detectors.py（MediaPipe Tasks）· worker.py（多进程推理）· simulator 兜底 │
└─────────────────────────────────────────────────────────────────────────────────────┘
```

**三条工程铁律**：
1. **离线优先**：无摄像头→模拟器；无大模型→规则脚本/规则建议；任何增强失败静默降级，主流程永不阻塞。
2. **隐私本地化**：画面仅在本地内存逐帧处理、当帧销毁，绝不存盘/上云；多端同步只传聚合指标。
3. **UI 秒开**：重依赖（mediapipe/opencv）全部延迟导入 + 后台线程加载，加载期用模拟数据占位，就绪后热切换。

## 二、视觉检测层（vision/）

### 2.1 模型与运行时
- **MediaPipe Tasks API（0.10+）VIDEO 模式**，两份轻量模型随仓库分发：
  - `face_landmarker.task`：468+10 面部关键点，`num_faces=1`，开启 `output_face_blendshapes` 输出 **52 维表情系数**（供情绪分析零成本复用）；
  - `pose_landmarker_lite.task`：BlazePose 33 点人体姿态。
- 实测环境：Python 3.14 + mediapipe 0.10.35 + opencv 4.13。

### 2.2 眨眼检测（EAR，Eye Aspect Ratio）
```
EAR = (|p2−p6| + |p3−p5|) / (2·|p1−p4|)
```
- 右眼关键点 `[33,160,158,133,153,144]`，左眼 `[362,385,387,263,373,380]`，双眼取均值；
- 状态机判定（`blink_counter.py`）：EAR < 0.21 连续 ≥2 帧，**或**闭眼墙钟时长 ≥0.10s（抗掉帧兜底），EAR 回升沿计一次眨眼；
- 频率折算（`metrics.py`）：60s 滚动窗口（分母下限 15s 防启动期虚高）+ EMA(α=0.15) 平滑。

### 2.3 用眼距离估算（单目视差测距）
```
distance = focal × 真实瞳距 / 像素瞳距
focal = (W/2) / tan(HFOV/2)        # 由水平视场角推导，与分辨率解耦，默认 HFOV=60°
```
- 优先用虹膜中心关键点 `468/473` 连线 + 人群平均瞳距 63mm；虹膜不可用时回退内眦 `133/362` + 32mm。

### 2.4 坐姿检测（CVA 颅椎角）
- 同侧「耳(7/8)—肩(11/12)」连线与水平线夹角 `atan2(dy,dx)`；可见度 <0.30 的一侧剔除，双侧按可见度加权；EMA(α=0.25) 平滑。CVA 越小=头越前倾（颈椎负荷越大）。
- 高低肩：双肩连线与水平夹角 >8° 提醒。

### 2.5 多进程推理（`worker.py`，`--mp-vision`）
采集+推理搬入 `multiprocessing`(spawn) 子进程，`Queue(maxsize=4)` 回传可 pickle 的 FrameSample，队列满丢最旧帧保实时性；15s 就绪超时/进程崩溃自动回退线程模式。

## 三、核心算法层（core/）

### 3.1 监测编排（`monitor.py`）
- `_CameraSampler` 独立采样线程 + `deque(120)` 批量 drain（不丢眨眼事件）；
- 每帧产出 **Snapshot**：`backend / face_present / blink_rate / blink_total / cva / distance / screen_time_min / continuous_use_min / next_break_in_sec / advices / mode / mood / persona_line / worst_severity / strict_alerts / emotion / guard_action / game_mode` 等；
- 台词节流：告警 18s / 夸奖 45s；恢复良好短暂 HAPPY 6s；无异常且离席 ≥40s 桌宠入睡。

### 3.2 表情情绪分析（`emotion.py`，零新模型）
52 维 blendshape → **13 个 FACS 动作单元(AU)**（如 au4=browDown 左右均值、au12=mouthSmile、au45=eyeBlink）→ Ekman 原型加权打分（例：positive = smile·1.25 + cheek·0.55 − frown·0.85；tired = blink·0.85 + jaw·0.80 + lidTight·0.60）→ 5 类标签 `positive/neutral/tired/stressed/negative`。
- 稳定性：AU 通道 EMA(α=0.25) + 中性基线 0.16 + 迟滞裕度 0.10 抗抖动；并列时按「疲惫>压力>低落>积极」仲裁。纯启发式规则，可解释、无需训练。

### 3.3 分级建议（`advice_engine.py`）
info/warn/alert 三级 × eye/posture/distance/break 四类：眨眼 <10 次/分 alert、<15 warn；CVA <45° alert、<50° warn；距离 <45cm；高低肩 >8°。**只做行为引导，不做医疗诊断（合规红线）。**

### 3.4 20-20-20 休息（`rest_timer.py`）
脸在计时、离座清零；满 20 分钟提醒远眺 6 米外 20 秒；连续 45 分钟强提醒。

### 3.5 极端用眼强制干预（`guardian.py`）
三条件**同时**满足并持续 20s 触发：连续用眼 >90min 且 距离 <35cm 且 眨眼率 <6 次/分；冷却 300s 防连发。两档强度（配置可选）：
- **soft（默认）**：全屏半透明遮罩 + 弥悠关怀台词 + 30s 强制倒计时；
- **hard**：`ctypes.windll.user32.LockWorkStation()` 真系统锁屏（非 Windows 自动降级 soft）。

### 3.6 桌宠人格与聊天（`persona.py` / `chat_engine.py`）
- **好感度养成**：0–100 六档（陌生/脸熟/朋友/在意/心动/深爱）；12 类关键词话题各配增量区间，重复同话题减半、单日上限 +25、离开 ≥2 天日衰减 2（下限 10），SQLite 跨会话持久化；
- **分层记忆**：短期=会话内 deque 6 轮×2；中期=SQLite 关键对话事件（|Δ好感|≥3）跨会话回注；
- **双模式 LLM 台词**：聊天模式（多轮对话）与非聊天模式（气泡台词后台扩充池，每场景 ≤8 条）共用「弥悠 D+A 身世」系统提示词；LLM 生成台词、离线规则判分（防刷分）；LLM 不可用时行为与纯离线完全一致。

### 3.7 大模型客户端（`llm_client.py`，零 SDK 依赖）
纯标准库 `urllib` 实现 OpenAI 兼容 `/chat/completions` 与 Ollama `/api/chat`；探测优先级：① 环境变量 OpenAI 兼容 API → ② 本地 Ollama:11434（可自动无窗拉起 `ollama serve`）→ ③ 无（30s 后复检）；`strip_think()` 剥离 DeepSeek-R1 思维链。聊天默认 `qwen2.5:7b-instruct`，复盘分析默认 `deepseek-r1:7b`。

## 四、健康报告体系

### 4.1 会话报告（`health_report.py`）
- **规则评分**（100 起扣）：眨眼 <10 扣 25 / <15 扣 12；CVA <45° 扣 25 / <50° 扣 12；距离 <45cm 扣 12；不良坐姿占比 ×40（上限 20）；
- **AI 行为洞察**：结构化指标喂 DeepSeek-R1 蒸馏模型产出个性化习惯分析与改善建议（超时 90s，失败回退规则建议），文末保留「非医疗诊断」免责声明。

### 4.2 可视化图表（`report_charts.py`，纯标准库内联 SVG）
四维雷达图（眨眼/距离/坐姿/时长节律）、三线趋势图、24 小时用眼热力图（绿→红渐变）、风险时段标注。零绘图库依赖，HTML 直接内嵌。

### 4.3 周报/月报（`period_report.py`）
`--report weekly|monthly` 聚合 7/30 天 SQLite 会话数据，自绘每日柱状+得分折线，跨周期 AI 洞察。

### 4.4 数据存储（`history.py`，SQLite 三表）
- `sessions`：时长/眨眼率/CVA/高低肩/距离/不良坐姿分钟/过近分钟/得分/主导情绪/情绪分布；
- `affection`：好感度单行状态（value/total_turns/daily_gain/daily_day）；
- `chat_events`：关键对话事件（保留 200 条）。

## 六、皮肤 AIGC 创作管线（弥悠官方皮肤）

**工具链**：本地 Stable Diffusion WebUI（anything-v5 动漫模型）+ API 自动化脚本（`/sdapi/v1/txt2img`、`/sdapi/v1/img2img`、ControlNet、ADetailer），全程本地推理。

**v4 管线（参考图高保真复刻 + 姿势差分）**：
1. **基准立绘**：官方参考图 → `img2img`（denoise 0.50）1.5 倍放大重绘（512×896 → 768×1344），提升清晰度同时锁定构图与服装；
2. **姿势差分**：`txt2img`（放开姿势自由度，按情绪写眼神/嘴唇/手部/身体四要素动作提示词）+ **ControlNet `reference_only`**（以基准立绘为参考锁定角色与服装风格，style fidelity 0.8）+ 高清修复（R-ESRGAN 4x+ Anime6B，1.5×）；
3. **细节修复**：ADetailer 双模型——`face_yolov8n.pt` 修脸（浅粉紫瞳+单颗淡美人痣）+ `hand_yolov8n.pt` 修手（防手指崩坏）；
4. **一致性控制**：固定种子 + 高权重服装提示词（如 `(black blouse with lace chest panel:1.5)`）+ 负面词封锁漂移方向（禁露肩/白袜/多颗痣/粉色描边等）；
5. **预抠图**：纯灰背景 + 四角八点泛洪（阈值 28）+ 背景均值封孔（d<45），直出 RGBA 透明立绘，`pet.py` 检测到自带 alpha 直接采用。

**历代要点**：v1 首发（txt2img 直出）→ v2 参考图重制（inpaint 定妆逐项攻坚）→ v3 白丝绝对领域/浅瞳/加高画布 → v4 图生图高保真+reference_only 姿势差分。

## 七、辅助工程能力

- **单实例互斥**（`main.py`）：Windows `CreateMutexW("Global\\ChenguanBrightEye_SingleInstance")`，错误码 183 时弹提示退出；
- **游戏勿扰**（`system_watch.py`）：ctypes 前台全屏检测（排除桌面窗口）自动静默；DDC/CI 显示器物理亮度守护（可选依赖 monitorcontrol，强制休息时调暗至 35%）;
- **多端同步**（`sync.py`）：局域网 HTTP 服务（端口 8765），`POST /api/usage`、`GET /api/summary`，X-Sync-Token 鉴权，仅传聚合指标；
- **四模式**（`modes.py`）：companion/strict/review/silent；严格模式 12s 升 1 级（0–3 级色阶递进）；
- **模拟器**（`simulator.py`）：合成眨眼率 17→6 次/分衰减、CVA 58°→40°、伪 blendshapes，支持 `--fast`，保证零硬件可演示；
- **打包**（`tools/build_exe.py`）：PyInstaller 单目录 + Inno Setup 中文安装向导（大模型可选随装、卸载程序归入安装目录）。
