> **华硕天选姬（ASUS TX Gaming 虚拟偶像桌宠）**的技术栈，并给出本项目
> 弥悠桌宠的候选迁移路线。**本文档只做调研结论，当前版本不动代码。**

## 一、天选姬是怎么做的（业界主流 = Unity + Live2D + Win32 API）

天选姬桌面版及同类商业桌宠（如各大厂虚拟偶像桌面端）的通用配方：

1. **渲染层：Unity + Live2D Cubism SDK**
   - 立绘由画师按部件分层（头发/眼/嘴/身体），Live2D 绑定网格与参数，
     实现呼吸、眨眼、视线跟随、口型、点击反馈等**骨骼级平滑动画**；
   - Unity 负责渲染循环、状态机（idle/被拖拽/被点击/说话）与资源管理。
2. **窗口层：Win32 API 改造 Unity 窗口为「透明、置顶、可点透」**
   - `SetWindowLong(GWL_EXSTYLE, WS_EX_LAYERED | WS_EX_TRANSPARENT)`：
     分层窗口 + 鼠标穿透；
   - `SetLayeredWindowAttributes(..., LWA_COLORKEY)` 或
     `DwmExtendFrameIntoClientArea`（DWM 全透明）：背景色抠掉只留角色；
   - `WS_EX_TOOLWINDOW` + 托盘图标：不占任务栏；
   - **全局鼠标钩子做"选择性点透"**：光标落在角色像素上时取消
     `WS_EX_TRANSPARENT`（可拖拽/点击），移出后恢复穿透——这是
     商业桌宠"既不挡操作、又能互动"的关键细节。
3. **功能层**：C#（Unity 脚本）驱动台词/换装/联动硬件灯效；配置与资源热更新。

## 二、候选技术栈横向对比

| 路线 | 代表项目 | 动画上限 | 常驻内存 | 与本项目衔接 |
| --- | --- | --- | --- | --- |
| **Unity + Live2D**（天选姬同款） | iwakura-lain/desktop-pet（Unity 2022.3 LTS + URP + Live2D） | ★★★★★ 骨骼级 | ~800MB（空场景即高） | 需 C# 重写互动逻辑，经 HTTP/WebSocket 读监测数据 |
| **Electron + PIXI + pixi-live2d-display** | git2968/Desktop-pet（Electron 33 + React）、OpenDesktopPet（含 AI 对话+TTS 口型） | ★★★★ Live2D 全支持 | <100MB（Canvas 渲染） | JS 生态，与「Wails/Tauri UI 升级路线」同栈可复用 |
| **原生 C++ (Win32 + Direct2D)** | 传统桌宠（如 Shimeji 类） | ★★★ 帧动画 | ~50MB 最省 | 开发成本最高，不建议 |
| **现状：tkinter 差分立绘** | 本项目 `ui/pet.py` | ★★ PNG 差分切换 | 已含在主进程 | 零依赖离线铁律的最优解（演示期保留） |

> 内存数据为社区项目实测量级：Live2D 单角色约 300MB 显存开销，Unity 空场景
> 常驻 ~800MB，Electron+Canvas <100MB，原生 C++ ~50MB。

## 三、对弥悠的迁移建议（分阶段，与皮肤系统衔接）

1. **演示期（现在）**：保留 tkinter 差分立绘——离线零依赖、任何机器可跑，
   这是评审场景的刚需；皮肤系统（`docs/皮肤系统设计方案.md`）继续按差分
   PNG 设计，画师资产（normal/happy/pout/…）**未来可直接升级为 Live2D 部件源图**。
2. **产品化第一步（推荐）**：**Electron/Tauri + pixi-live2d-display** 独立
   桌宠进程——性价比最高：Live2D 全能力、<100MB、Web 技术栈与
   `docs/UI技术栈升级路线.md`（Wails/Tauri）同向；监测核心仍是 Python，
   经本地 WebSocket 推送 `Snapshot`（复用 `core/sync.py` 的服务化思路），
   桌宠只做「显示 + 互动」，**天然跨语言解耦**。
3. **产品化第二步（预算充足）**：委托 Live2D 建模师制作弥悠模型
   （分层 PSD → Cubism 绑定，市场价约数千~数万元/体），如需硬件联动
   （对齐天选姬的灯效生态）再评估 Unity 路线。
4. **Win32 细节清单（无论选哪条路都要做）**：透明分层窗口、选择性点透
   （像素命中检测）、托盘常驻、开机自启、多显示器 DPI 适配。

> 一句话总结：**天选姬 = Unity + Live2D + Win32 透明窗口三件套；弥悠的
> 最优演进是先保 tkinter 演示，再走 Electron/Tauri + Live2D 独立桌宠进程，
> 与 UI 升级路线、皮肤系统共用同一套 Web 栈投资。**
