# 皮肤专项功能 · 设计方案（v1.12 规划，未实施）

> 目标：支持用户在软件界面**直接上传桌宠形象（立绘 PNG）与人设 soul.md**，
> 不改代码即可更换"自己的桌宠"。本方案为设计蓝图，实施排期到下个版本。

## 一、皮肤包结构（目录即皮肤）

```
data/skins/<皮肤名>/
├── soul.md              人设档案（人格提示词来源，格式同 docs/弥悠人设.md）
├── skin.json            元信息：显示名 / 作者 / 台词风格标签 / 主题色(可选)
└── sprites/             差分立绘（可缺省，缺省则用矢量兜底）
    ├── normal.png       常态（必需）
    ├── happy.png / pout.png / angry.png / sleepy.png   （可选，缺省回退 normal）
    └── blink.png        眨眼差分（可选）
```

- 内置皮肤「弥悠」打包为第一个官方皮肤（当前 `wenna_*.png` + 弥悠人设.md 平移进来）；
- 一切字段可缺省：只传一张 normal.png 也能用；连 sprites 都没有就走程序化矢量兜底
  ——延续「任何输入残缺都安全降级」铁律。

## 二、界面交互（零门槛上传）

1. 桌宠右键菜单/仪表盘新增「🎨 皮肤中心」窗口：
   - 皮肤列表（预览缩略图 + 显示名 + 作者），点击即切换、即时热加载；
   - 「＋导入皮肤」按钮：`filedialog` 选文件夹或 zip → 校验后拷入 `data/skins/`；
   - 「上传立绘」「上传 soul.md」两个单项替换入口（对当前选中皮肤做局部覆盖）。
2. 热加载路径：
   - 立绘 → 复用现有 `ui/pet.py` 立绘缓存机制，切换时清缓存重新 floodfill 抠图；
   - soul.md → 解析「性格/台词/拟声词」段落拼进 `chat_engine` 人格提示词与
     `persona.py` 台词池；解析失败回退弥悠默认人设。

## 三、soul.md 约定格式（宽松解析）

按二级标题识别，全部可缺省：`## 性格`（→ LLM 人格提示词）、`## 台词`
（→ 离线台词池，按「眨眼/距离/坐姿/休息」四场景行首标签解析）、
`## 拟声词`、`## 称呼`。识别不到的内容整体作为背景设定附在提示词末尾。

## 四、安全与合规

- 仅接受 PNG（Pillow/tk 可解码校验）与 UTF-8 文本；单文件 ≤10MB；
- soul.md 注入 LLM 前拼接固定的**防出戏/医疗合规系统规则**（不可被皮肤覆盖）：
  不诊断不开药、不脱离健康助手职责；
- 好感度/防刷分/强制干预等核心逻辑与皮肤无关，皮肤只影响「外观与说话方式」。

## 五、实施拆分（建议两步走）

| 阶段 | 内容 | 涉及文件 |
| --- | --- | --- |
| P1 | 皮肤包加载器 + 右键切换 + 立绘热加载（先支持内置弥悠 + 用户文件夹） | 新 `core/skin.py`、改 `ui/pet.py` |
| P2 | 皮肤中心窗口（列表/导入/单项上传）+ soul.md 解析注入 | 新 `ui/skin_center.py`、改 `chat_engine.py`/`persona.py` |

与本版已落地的**双主题系统（ui/theme.py）**天然衔接：skin.json 的可选主题色
字段未来可直接注册为第三套配色 token。
