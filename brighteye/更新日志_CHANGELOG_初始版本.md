## v1.7.0-demo — 新增「和文乃聊天」galgame 模式（自动对话 + 好感度养成）（2026-06-24）

**主题：右键桌宠新增「💬 聊天模式」，打开一扇 galgame 风对话窗——立绘 + 经典底部对话框 +
打字机逐句（伪多条对话）+ 顶部好感度条。性格＝芹泽文乃（重度傲娇、言不由衷、猫系），
玩家自由打字、文乃自动回话（不给选项分支），并「自主」判定好感度增减、可像 galgame 一样随时查看。**

### 功能点
- **新增聊天引擎 `core/chat_engine.py`（离线优先，零联网即可演示）**：
  - `ChatEngine.respond()` 按关键词把玩家输入分类（问候/表白/夸奖/道谢/护眼/吐槽/疲惫/告别…），
    每类回 1~3 句傲娇台词（**伪多条对话**），并由文乃**自主给出好感度增量**（-10~+12）；
  - **好感度系统**：0~100 分、六级（陌生→脸熟→朋友→在意→心动→深爱），同话题反复刷分自动衰减；
    `level_of()` 输出等级名/心数/进度，供 UI 可视化（像 galgame 恋爱进度）；
  - **自动对话**：`idle_auto()` 在长时间无输入时让文乃主动开口；`greeting()` 按好感度给不同开场白；
  - **预留大模型钩子**：设 `BRIGHTEYE_LLM_KEY` 可接入 LLM（含人格系统提示词模板、防出戏黄金法则），
    未配置时安全回退离线脚本——演示不依赖任何 API。
- **新增聊天窗口 `ui/chat.py`（galgame 风）**：
  - **立绘**（复用 `assets/pet` 加载/抠图/LANCZOS 缩放）按情绪切换（开心/嘟嘴/生气…），缺图用占位卡；
  - **底部对话框**＝半透明深色 + 翠绿描边 + 「文乃」名字牌 + **打字机逐字输出**，点「继续 ▼」/空格推进；
  - **顶部好感度条**＝心形（♥/♡）+ 等级名 + 进度条 + 本次增量浮字，实时刷新；
  - 底部输入框自由打字、回车/「发送」即对话；约 18s 无输入文乃自动开口。
- **右键菜单接入（`ui/pet.py`）**：新增回调 `on_chat` 与菜单项「💬 聊天模式（和文乃说说话）」；
  `ui/app.py` 以**单例**方式打开/聚焦聊天窗，并与桌宠共享同一 `ChatEngine`（好感度跨窗一致）。
- 版本号 `1.6.0-demo → 1.7.0-demo`。

## v1.6.0-demo — 可爱气泡 + 全情绪眨眼 + 天选姬式桌宠对标（2026-06-23）

**主题：对话框升级为二次元可爱风（圆角奶白卡 + 彩色描边 + 小爱心 + 圆尾、按真实像素宽换行不再溢出）；
眨眼支持「全情绪差分 + 通用兜底」且整体频率调低；并对标华硕「天选姬」式桌宠形态，
明确本项目=天选姬式陪伴 + 独有用眼提醒。**

### 更新点
- **气泡重做为二次元可爱风（`ui/pet.py::_draw_bubble`，修「文字超框 / 气泡丑」）**：
  - **按真实文本像素宽换行**：新增 `_wrap_by_px()` 用 `tkfont.Font.measure` 逐字测量，
    配合字体缓存 `_bubble_font()`，中英文混排都不会超出气泡边界（修掉文字溢出）；
  - **圆角奶白卡 + 彩色描边 + 朝下圆尾 + 顶部小爱心**：`_round_rect_pts()` 生成圆角多边形，
    奶白偏粉底 `#FFF3FA` + 情绪色描边 + 轻微贴纸阴影，整体更可爱、更贴合二次元；
  - 气泡宽高随文本与缩放 `sc` 自适应，居中悬于头顶、尾巴指向文乃，不挡脸。
- **眨眼：全情绪差分 + 通用兜底 + 频率调低（修「眨眼帧只在部分情绪有、动作突兀 / 太频繁」）**：
  - 新增按情绪的闭眼差分约定 `BLINK_FILES`（`wenna_<mood>_blink.png`）与挑选逻辑 `_blink_key()`：
    **该情绪专属闭眼帧 → 通用 `wenna_blink.png` → 保持睁眼**，逐级回退，任何情绪眨眼都不突兀；
  - 眨眼**频率调低**：约每 4~9 秒眨一次（`_next_blink` 节律放宽），更自然不闹腾；
  - 睡眠态仍保持闭眼睡颜、不插入眨眼帧。
- 版本号 `1.5.0-demo → 1.6.0-demo`；`assets/pet/README.txt` 增补各情绪眨眼差分说明。

### 调研与定位（天选姬对标）
- 调研 GitHub 主流桌宠（NyaDeskPet / Live2DPet / EasyLive2D-Live2DMascot / Deskpet 等）：
  「天选姬」类桌宠的共性实现＝**Live2D/立绘角色 + 透明无边框置顶窗 + 待机微动 + 拖拽/右键 + 情景气泡**。
- 本项目桌宠已具备同一形态（透明置顶窗、立绘差分、待机呼吸/眨眼、拖拽缩放、头顶气泡），
  **差异化优势**＝在陪伴之上叠加独有的**非接触用眼健康提醒**（眨眼/距离/坐姿/久用眼 → 情绪化催促），
  即"天选姬式陪伴 + 护眼管家"。后续如需口型同步/语音，可接 Live2D 路线（见 README 调研链接）。

## v1.5.0-demo — 立绘接入打磨：抠背景 / 气泡不挡脸 / 平静常态（2026-06-23）

**主题：导入 galgame 立绘 PNG 后的六项体验修复——背景自动抠除、对话框移到头顶不再挡脸、
睡眠不再眨眼、去掉突兀光环、平静(NORMAL)作为常态表情、窗口按立绘宽高比自适应不裁切。**

### 更新点
- **立绘背景自动抠除（`ui/pet.py`，新增 Pillow 路线）**：
  - 新增 `_pil_remove_bg()`：从四角 `ImageDraw.floodfill` 判定连通的纯色背景并置为透明，
    前景（文乃本体）保持不透明——修掉「PNG 背景方框非常突兀」；
  - 已自带透明像素的图（alpha 含 0）原样保留，不重复抠图；
  - 缺 Pillow 时自动退回 `tk.PhotoImage`（不抠图、整数倍缩放），demo 仍可跑。
- **高质量缩放 + 窗口自适应（修「图片大小不适配、元素被裁切」）**：
  - 新增 `_pil_to_photo()`：按目标高度等比 **LANCZOS** 缩放（取代整数倍 subsample/zoom）；
  - 窗口/画布尺寸改为**按立绘宽高比计算**（`art_w0 = art_h × 立绘宽/高`），
    顶部留出气泡区、侧/底少量留白，立绘元素不再被窗口边缘切掉。
- **对话框移到头顶（`_draw_bubble`，修「对话框挡住图片」）**：
  气泡改为**顶部预留区、水平居中、尾巴朝下指向文乃**，并随缩放 `sc` 同步缩放，不再覆盖脸/身体。
- **睡眠不再眨眼（`_draw_sprite`，修「睡眠模式还会眨眼」）**：
  睡眠态下不再插入 `blink` 帧，保持闭眼睡颜。
- **去掉脚下光环（修「底部光环突兀」）**：立绘模式不再绘制情绪光环椭圆，画面更干净。
- **平静(NORMAL)作为常态表情（`core/persona.py` + `core/monitor.py`，修「文乃一直高兴」）**：
  - `Persona.mood_for` 健康分支由 `HAPPY` 改为返回 `NORMAL`（平静基准）；
  - `monitor` 新增「恢复→开心」瞬态：仅在**刚从不良用眼恢复**时短暂(约 6s)切到 `HAPPY`，
    随后回到平静，避免一直咧嘴笑、显不出平静状态。
- 版本号 `1.4.0-demo → 1.5.0-demo`。

### 兼容性
- Pillow 为**可选**依赖：有则抠图 + 高质量缩放，无则自动降级；矢量兜底完全不受影响。

## v1.4.0-demo — CVA 测量提精 + 桌宠可缩放 + 安静自动入睡（2026-06-22）

**主题：三项打磨——颅椎角(CVA)算法更准、文乃形象可拖动缩放并右键选大小、
降低唠叨频率并让文乃在安静一段时间后真正进入睡眠形象。**

### 更新点
- **颅椎角(CVA)测量提精（`vision/detectors.py`）**：
  - **顶点改用「双肩中点」近似 C7**（脊柱中线），取代原先的「单侧肩关节」——
    肩关节偏身体两侧会让水平分量系统性偏大、角度被低估；改用中线后，端正直坐时
    耳≈在 C7 正上方→角度≈90°，更贴近 CVA 解剖定义（实测：端正 90° / 轻度前倾 66° / 明显前倾 35°，单调可分）。
  - **可见度门控**：仅纳入「耳+肩」均足够可见的一侧（阈值 `cva_vis_floor`），
    侧脸/头发/手遮挡时自动剔除该侧，避免拉偏；两侧都不可靠时回退尽力估计。
  - **时间指数平滑(EMA)**：新增 `cva_smooth`，抑制逐帧关键点抖动，读数更稳。
- **桌宠文乃可缩放（`ui/pet.py`）**：
  - **右键菜单新增「形象大小」**：小75% / 标准100% / 大140% / 超大180% + 放大/缩小；
  - **鼠标滚轮无级缩放**（0.6×～2.2×），缩放时窗口/画布自适应并保持左上角位置；
  - 立绘按当前缩放整数倍重采样并缓存；矢量兜底整体缩放（几何+线宽+字号一并缩放）保持清晰；
  - 原有「按住拖动移动位置」保留。
- **降低唠叨 + 安静自动入睡（`core/monitor.py` + `core/persona.py` 节奏）**：
  - 台词复述间隔放宽（同类问题 8s→18s，状态良好夸奖 24s→45s），减少刷屏；
  - 新增 `idle_sleep_sec`（默认 40s）：**持续无异常超过该时长，文乃犯困入睡并停止说话**，
    修掉「话太多导致睡眠形象从不出现」的问题；出现异常或离座会唤醒并重置计时。
- 版本号 `1.3.0-demo → 1.4.0-demo`。

### 新增可调参数（`config.py`）
- `cva_smooth=0.25`、`cva_vis_floor=0.30`、`idle_sleep_sec=40.0`，均集中可现场调参。

## v0.3.0 — 三项核心算法精度优化

**主题：在不牺牲准确性的前提下，提升实时性与测量精度（真实摄像头实测验证）。**

### 更新点
- **眨眼频率改为实时滚动统计**：由"累加平均"改为 **30s 滚动窗口 + 指数平滑（EMA）**，
  响应更灵敏且抑制边缘抖动（`metrics.blink_rate_realtime`）。
- **用眼距离测算重做**：改用**虹膜瞳距（iris 468/473）** 估距，焦距由**水平视场角(FOV)推导**（分辨率无关），
  无虹膜点时回退内眼角；实测约 **54.1 cm**（`vision/detectors.py`）。
- **颅椎角(CVA)测算重做**：改为**左右双侧、按关键点可见度加权平均**，抗单侧遮挡；实测约 **51.6°**。
- 指标缓冲队列优化（`metrics.py`），降低内存占用。

## v0.1.0 — 立项与核心监测原型

**主题：跑通"监测 → 实时建议 → 健康报告"全链路，任意环境可演示。**

### 功能点
- **非接触式视觉监测**：普通摄像头实时分析**眨眼频率、用眼距离、坐姿（颅椎角 / 高低肩）、屏幕时长**。
- **MediaPipe Tasks API 后端（`vision/detectors.py`）**：FaceLandmarker（478 点含虹膜）+ PoseLandmarker。
- **优雅降级**：无摄像头 / 无依赖时自动回退**内置模拟器**（`core/simulator.py`），保证答辩可演示。
- **规则化分级建议引擎（`core/advice_engine.py`）**：info/warn/alert 三级，结合 **20-20-20 护眼法则**；
  合规红线——只做行为引导与就医提示，不做诊断与处方。
- **健康评分 + HTML 报告（`core/health_report.py`）**：会话结束生成可视化报告，可直接截图进 PPT。
- **Tkinter 实时仪表盘（首版）** + 阈值集中可配置（`config.py`）。
